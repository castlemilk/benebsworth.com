## Selection heuristic — which to write next?

When choosing the next article from this backlog, score on three axes (1–5 each, pick the highest sum):

1. **Reusable lab effects available** (1 = none, 5 = multiple existing effects) — 5 gives a fully interactive post for free
2. **Strong visual / interactive payoff** (1 = pure prose, 5 = the topic demands a diagram) — interactive posts get 5–10× the engagement
3. **Personal conviction** (1 = "could write", 5 = "have to write this") — authenticity reads. Topics you'd defend in a conversation

Total ≥ 12 → write it. Total < 8 → wait.

## Component gaps

The blog's MDX component library is missing:
- `Equation` — display math via KaTeX/MathJax. Several backlogs (PLL, modulation, FFT, normal modes) need it.
- `CodeSandbox` or `CodePen` embed — for live-edit examples
- `ComparisonTable` — side-by-side feature comparison (current `prose` tables work but aren't styled)
- `Timeline` — for "X → Y → Z" history sections
- `BoxCallout` variants — there's `<Callout type=...>` but no compact "key insight" pill, no warning with a specific icon, no quote-block style
- `Section` — anchor for sub-sections within a long post (with a small visual divider)
- `ProgressiveImage` — currently uses `<img>`, no blur-up placeholder

Backlog items that need **at least one new component** are marked in the categories above.
